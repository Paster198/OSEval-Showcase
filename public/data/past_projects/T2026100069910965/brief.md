OSKernel2026 是一个为 2026 年 OS 内核实现竞赛从零构建的 C 语言内核，同时支持 RISC-V 64 与 LoongArch 64 架构。该项目以真实二进制兼容性为核心特点，能够加载并运行未经修改的静态链接 musl 或 glibc BusyBox 二进制文件，通过让真正的 BusyBox shell 解释测试脚本实现逐字节的输出匹配，而非在内核中模拟命令。

内核实现了 Sv39 多级页表（RISC-V）及 LoongArch 软件 TLB 管理，提供完整的写时复制 fork、按需页面分配、mmap 文件映射与共享内存等内存管理机制。文件系统层以 ramfs 为基础提供可写存储，仅在内存中构建 /dev、/proc、/bin 等目录，并实现只读 ext4 驱动以读取测试数据，支持 extent 树的解析与 LRU 风格缓存。系统调用覆盖约 90 个 Linux 兼容调用，包括文件 I/O、进程创建（clone/clone3）、管道、本地回环 socket 以及 System V 共享内存等。此外，内核通过路径别名与解释器解析自动适配 musl/glibc 库路径，降低文件系统依赖。

该项目在调度器、信号处理与真实网络设备驱动方面仍属初期实现，但整体上在 ABI 兼容性与文件系统支持层面展现了较强的工程交付能力。
