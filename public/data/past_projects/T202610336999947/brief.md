ArceOS 是一个基于 Rust 语言开发的组件化 Unikernel 操作系统，支持 x86_64、RISC-V 64、AArch64 及 LoongArch64 四种硬件架构。其技术定位是高度可裁剪的内核构建框架：通过 Cargo feature 系统在编译期按需选择内核模块，生成与上层应用紧密耦合的最小化系统镜像，兼顾性能、体积与灵活性。

项目核心特点包括：完整的硬件抽象层覆盖内存初始化、页表操作、中断管理与 Per-CPU 数据；多架构统一的驱动框架支持 PCI/MMIO 总线探测，集成 VirtIO 块设备、网络与图形驱动；拥有独立的虚拟内存与物理内存管理子系统，提供 slab、buddy、TLSF 三种分配器；任务管理支持 FIFO、Round-Robin 和 CFS 三种在编译期可切换的调度策略，具备 SMP 启动、等待队列与定时器机制；文件系统通过 VFS 挂载点管理 FAT、RamFS 与模拟的 proc/sys 结构；网络栈基于 smoltcp 实现 TCP/UDP 协议及 DNS 查询。兼容层方面，arceos_posix_api 实现了广泛的 POSIX 接口，axlibc 将其封装为 C 标准库，可支撑类 Unix 应用的直接运行。

该项目在架构设计上具有多项亮点：通过链接段与命名空间抽象实现资源隔离的 axns 框架，可同时支持 unikernel 的全局共享模式与宏内核风格的进程隔离模式；驱动与调度器的双模式/可替换设计，在编译期即可完成性能与灵活性的权衡；自底向上的多层级 API 为从裸机开发到 C/POSIX 应用提供了广泛的兼容能力。
