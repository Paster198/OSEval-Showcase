GCore 是一个使用 Rust 语言从零构建的类 Unix 操作系统内核，定位为面向 RISC-V 64 与 LoongArch 64 双架构的通用单体内核。它通过硬件抽象层统一了两套完全不同的页表、陷阱和 MMU 机制，能够在 QEMU 及多款真实开发板上运行。项目实现了覆盖进程管理、内存管理、文件系统、信号、网络和同步原语等主要子系统的约 139 个系统调用，具备运行 BusyBox、Lua 等复杂用户程序的能力。

GCore 的核心亮点在于文件系统与内存管理的工程深度。其文件子系统基于统一的 VFS 框架，同时支持 ext4 和 FAT32 文件系统，并实现了完整的 ext4 extent 树算法，包括 extent 分裂、合并与分配等生产级特性。内存管理方面，项目构建了写时复制、mmap、Zram 压缩、磁盘 Swap 以及 OOM handler 组成的三级内存回收体系，能够在资源紧张时主动回收或换出页面。网络栈基于 smoltcp 提供了 TCP/UDP/ICMP 及 Unix 域套接字支持。信号系统实现了 64 个实时信号、sigaltstack 和 sigreturn trampoline。这些设计体现了对 Linux 内核机制的深入理解，在同类竞赛或教学项目中具有较高完成度与技术创新性。
