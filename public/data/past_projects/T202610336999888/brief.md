OrayS 是一个基于 ArceOS 组件化内核框架构建的 Linux 兼容操作系统内核，定位于操作系统内核竞赛场景，面向 RISC‑V64 和 LoongArch64 等架构提供轻量、高兼容的用户态二进制运行环境。

该项目在 ArceOS 原有的硬件抽象、内存管理、任务调度、文件系统和网络栈之上，构建了约 37,000 行 Rust 代码的 Linux ABI 兼容层，实现了超过 230 个系统调用，覆盖文件 IO、进程管理、内存映射、信号、管道与 System V/POSIX IPC、socket、epoll、定时器等核心类别。其进程模型完整支持 fork/vfork/clone 的写时复制和共享映射语义，以及包含脚本解释器和 PIE 的 execve 加载，能够在无真实 Linux 内核的情况下直接运行 BusyBox、LTP 等基准负载。

OrayS 在实现上展现了多项关键技术亮点：精细化建模的进程控制块，包含超过 50 个属性字段以准确呈现 Linux 进程的可观测行为；per‑process 虚拟文件系统元数据层，使权限、扩展属性、符号链接等操作无需真实文件系统即可正确响应；CFS 调度器直接复用 Linux 内核的 nice 到权重映射表，保证了调度行为的语义兼容；面向 RISC‑V64 的完整信号栈帧布局与内联 trampoline 代码，实现了从硬件异常到用户信号投递的闭合链路；以及内嵌的 LTP 测试运行器、内存泄漏检测和按架构分离的黑名单机制，为快速验证和回归测试提供了完整基础设施。整体上，OrayS 在保持 ArceOS 轻量化优势的同时，通过高度实用的 ABI 兼容层，在有限的代码规模内达到了较强的 Linux 用户态兼容能力。
