# OS 内核项目技术画像与评估报告

## 一、 项目基本信息

- **项目名称**：SystemNQB（基于 `uname` 系统调用返回值推断）
- **目标架构**：RISC-V 64位（RV64），采用 Sv39 虚拟内存模式
- **实现语言**：C 语言、RISC-V 汇编语言
- **代码规模**：约 48 个源码文件，总计约 7583 行代码（包含内核、用户态程序及 mkfs 工具）
- **生态归属**：操作系统内核比赛（oscomp）内核赛道参赛作品
- **核心特点**：
  - 采用 Linux RISC-V 兼容的系统调用 ABI。
  - 集成 FAT32 文件系统以替代原生简单日志文件系统。
  - 支持 SBI HSM（Hart State Management）扩展进行多核动态启动。
  - 构建时通过脚本自动生成系统调用分发表。

## 二、 实现的子系统与功能

项目实现了以下核心子系统与功能：
1. **启动与多核管理**：基于 SBI 规范的引导流程，支持通过 HSM 扩展动态启动多个 CPU 核心。
2. **内存管理**：基于空闲链表的物理页分配器，Sv39 三级页表虚拟内存管理，用户与内核地址空间隔离。
3. **进程管理**：静态进程表管理，进程创建（fork/clone）、退出、等待（wait4），基于轮转的简单调度器，上下文切换。
4. **中断与异常处理**：用户态与内核态陷阱处理，基于 Trampoline 机制的页表切换，PLIC 外部中断分发，SBI 定时器中断处理。
5. **系统调用接口**：兼容 Linux RISC-V ABI 的系统调用分发机制，实现 40 个系统调用中的 33 个。
6. **文件系统**：FAT32 文件系统适配，包含 BPB 解析、FAT 链管理、短文件名目录项处理、块 I/O 缓存（LRU）。
7. **文件与管道**：全局文件描述符表管理，标准管道（Pipe）读写与阻塞，ELF64 可执行文件加载。
8. **设备驱动**：VirtIO MMIO 块设备驱动，UART 16550 串口驱动，PLIC 中断控制器配置。
9. **同步机制**：基于原子操作的自旋锁（支持中断嵌套禁用），基于自旋锁与睡眠机制的睡眠锁。
10. **用户态环境**：极简 init 进程，基础 C 标准库子集（字符串操作、格式化输出、内存分配），系统调用桩代码生成。

## 三、 子系统实现完整度与细节分析

### 3.1 启动与入口子系统
- **完整度**：90%（基准：完成 SBI 引导与多核初始化）
- **实现细节**：入口汇编代码从 SBI 接收 hartid，计算独立栈地址并关闭分页。C 语言入口完成控制台、内存、进程、中断、文件系统等初始化。通过 SBI HSM 扩展检测并启动处于停止状态的非引导核。
- **优点**：SBI 集成规范，多核启动逻辑清晰，链接器脚本对 Trampoline 段有严格的页边界对齐断言。
- **缺点**：缺少对 SBI HSM 扩展返回值的错误检查，若多核启动失败可能导致系统状态不一致。

### 3.2 内存管理子系统
- **完整度**：75%（基准：实现基础物理分配与虚拟映射）
- **实现细节**：物理页分配器采用单一全局自旋锁保护的空闲链表，管理 128MB（0x80000000 至 0x88000000）物理内存。虚拟内存采用 Sv39 页表，内核页表映射 UART、VirtIO、PLIC 及内核代码/数据段。用户页表支持动态增长与缩减，`fork` 时执行完整的地址空间复制。
- **优点**：内存分配与释放时通过填充特定字节（0x05 和 0x01）辅助检测未初始化访问与悬空引用。
- **缺点**：物理分配器无 per-CPU 优化，高并发下锁竞争严重；未实现写时复制（CoW）机制；缺失 `mmap`/`munmap` 支持，无法进行内存映射文件操作。

### 3.3 进程管理子系统
- **完整度**：70%（基准：实现基础进程生命周期与调度）
- **实现细节**：采用大小为 64 的全局静态进程表。进程结构体包含状态、页表、陷阱帧、文件描述符表及当前工作目录路径字符串。调度器采用线性扫描的简单轮转算法（Round-Robin），时钟中断触发 `yield` 让出 CPU。上下文切换保存/恢复 14 个 callee-saved 寄存器。
- **优点**：进程结构针对 FAT32 进行了合理改造（使用路径字符串替代 inode 引用作为 cwd）；`clone` 系统调用支持指定新栈地址。
- **缺点**：调度器无优先级与时间片管理；`wait4` 的 `options` 参数未实际使用；`clone` 忽略了 `flags` 参数，无法实现线程级别的资源共享（如 `CLONE_VM`）；缺乏信号机制。

### 3.4 中断与异常子系统
- **完整度**：85%（基准：实现完整的陷阱分发与设备中断处理）
- **实现细节**：通过 Trampoline 机制实现用户态与内核态的页表切换与寄存器保存。`usertrap` 分发系统调用、设备中断与异常。设备中断通过 PLIC 分发至 UART 或 VirtIO 驱动。定时器中断通过 SBI 设置下一次触发，仅在 CPU 0 上递增全局 `ticks`。
- **优点**：Trampoline 设计保证了陷入与返回时页表切换的原子性与安全性。
- **缺点**：未实现页错误（Page Fault）处理，非法指令等异常仅标记进程为 killed 而无详细恢复或信号传递机制。

### 3.5 系统调用子系统
- **完整度**：65%（基准：实现声明的 Linux 兼容系统调用）
- **实现细节**：通过 `a7` 传递系统调用号，构建时利用 `sed` 脚本从宏定义自动生成系统调用分发表。在声明的 40 个系统调用中，完整实现 33 个，5 个返回 -1（`linkat`, `umount2`, `mount`, `munmap`, `mmap`），2 个部分实现（`times`, `gettimeofday`）。
- **优点**：自动生成机制降低了分发表的维护成本；核心文件与进程操作兼容 Linux ABI。
- **缺点**：`mmap`/`munmap` 的缺失严重影响依赖内存映射的应用运行；`times` 返回硬编码固定值；`gettimeofday` 基于 `ticks` 估算且基准值硬编码，精度与准确性不足。

### 3.6 文件系统子系统
- **完整度**：60%（基准：实现 FAT32 基础读写与目录操作）
- **实现细节**：解析 FAT32 BPB 获取文件系统参数，要求簇大小等于 4096 字节。inode 结构改造为使用父目录首簇号与目录项编号定位。实现 FAT 链遍历与空闲簇分配。块 I/O 缓存采用 30 块大小的 LRU 双向链表。
- **优点**：成功将原生文件系统替换为 FAT32，支持标准短文件名格式，适配外部 SD 卡镜像。
- **缺点**：日志系统被完全禁用，无崩溃恢复能力；不支持长文件名（LFN）；`mkfs` 工具仍生成原生格式镜像，与内核 FAT32 实现不匹配；不支持多 FAT 表镜像写入；`iupdate` 不支持簇重新分配。

### 3.7 文件、管道与执行子系统
- **完整度**：80%（基准：实现标准文件描述符管理与 ELF 加载）
- **实现细节**：全局文件表大小为 100。文件结构新增 `path` 字段以支持 `*at` 系列相对路径系统调用。管道采用 512 字节缓冲区与自旋锁保护。`exec` 加载 ELF64 文件，解析 `PT_LOAD` 段，分配 2 页用户栈并构造 `argc`/`argv`。
- **优点**：文件描述符与路径管理逻辑清晰，ELF 加载器支持段权限转换。
- **缺点**：ELF 加载器不支持解释器（Interpreter）与动态链接；`execve` 忽略环境变量参数 `envp`。

### 3.8 设备驱动子系统
- **完整度**：85%（基准：实现块设备与串口基础驱动）
- **实现细节**：VirtIO 块设备驱动支持 8 个描述符的队列，读操作同步阻塞，写操作异步中断完成。UART 16550 驱动支持 32 字节环形缓冲区的异步发送与中断接收。
- **优点**：VirtIO 驱动实现了多块操作与异步写入，提升了 I/O 吞吐量。
- **缺点**：缺少网络驱动支持。

### 3.9 同步机制子系统
- **完整度**：90%（基准：实现基础互斥与睡眠同步）
- **实现细节**：自旋锁使用 GCC 内置原子操作，结合 `push_off`/`pop_off` 实现中断禁用与嵌套支持。睡眠锁基于自旋锁与 `sleep`/`wakeup` 机制实现。
- **优点**：死锁检测机制完善，记录持有锁的 CPU 或进程 PID。
- **缺点**：缺乏读写锁、互斥量（Mutex）及条件变量等高级同步原语。

## 四、 OS 内核整体实现完整度

以“完整实现所声明的 Linux 兼容系统调用及现代操作系统基础子系统”为基准，通过对各子系统完整度进行加权计算（权重分配：系统调用 20%，内存管理 15%，进程管理 15%，文件系统 15%，启动与入口 10%，中断与异常 10%，文件与管道 5%，设备驱动 5%，同步机制 3%，用户态 2%），该 OS 内核的整体实现完整度约为 **73%**。

## 五、 动态测试设计与结果

**未进行动态测试**。
由于当前分析环境中缺失 RISC-V 交叉编译工具链、GNU Make 构建工具以及 QEMU 模拟器，无法完成内核的编译、链接与启动运行。本报告的所有评估与结论均基于对 7583 行源代码的静态审查与逻辑推演。

## 七、 总结评价

该项目是一个面向操作系统内核比赛的 RISC-V 64 位内核作品，核心工作集中在将基础教学操作系统架构改造为兼容 Linux RISC-V ABI 并适配 FAT32 文件系统的竞赛环境。

项目的优势在于工程改造的合理性：系统调用分发表的自动化生成降低了维护成本，SBI HSM 多核启动与 VirtIO 异步 I/O 的实现展现了对底层硬件规范的掌握，FAT32 的引入有效解决了比赛测试套件的镜像兼容问题。

项目的主要局限性体现在功能完整性与高级特性的缺失。`mmap`/`munmap` 系统调用的完全缺失是重大缺陷，将导致大量依赖内存映射的现代应用程序无法运行。文件系统日志机制的禁用牺牲了数据一致性保障。进程调度、时间管理及信号机制的实现停留在最基础的水平，缺乏对复杂并发场景与高精度时间控制的支持。此外，`mkfs` 工具与内核文件系统实现的不匹配反映了工程交付时的细节疏漏。

总体而言，该项目完成了竞赛环境的基础适配任务，具备可运行的内核骨架与核心 I/O 能力，但在系统调用的覆盖广度、资源管理的高级特性以及工程细节的严谨性上仍有明显的提升空间。
