Chronix 是一个采用 Rust 语言编写的宏内核操作系统，核心目标是提供类 Linux 的兼容性并支持 RISC‑V 64 与 LoongArch 64 双架构。它以 async/await 作为内核调度模型的基础，将异步语义贯穿于系统调用、文件 I/O 和网络操作，形成统一的“异步优先”执行框架，从而在等待 I/O 时自然让出 CPU。内核规模约 59,000 行 Rust 代码，实现了完整的进程管理（含线程组、进程组、UID/GID）、页式虚拟内存（写时复制、按需分页、动态链接器加载）、覆盖 ext4、FAT32、tmpfs、devfs、procfs 和 pipefs 的完备 VFS 层，以及 TCP/ UDP 网络栈。信号子系统完整支持 64 种信号和先进的 futex（含优先级继承与 robust futex），并集成了 System V IPC 三大机制。硬件抽象层通过 trait 和条件编译清晰隔离架构差异，复用率超过 90%。该项目在 Linux 兼容性上表现突出，约 130 个系统调用已有完整实现，并能运行 busybox、LTP 等外部测试套件，适合作为探索现代 OS 内核设计与双架构工程的系统原型。
