Starry_fix 是一个基于 ArceOS 组件化 unikernel 框架构建的 Linux 兼容宏内核，使用 Rust 语言实现，目标是提供完整的 Linux ABI 兼容性。项目在原有 StarryOS 的基础上进行深度改进，通过模块化的系统调用分发器实现了超过 200 个 Linux 系统调用，涵盖文件 I/O、进程管理、信号、网络套接字、System V IPC 及 epoll 等关键机制。内核采用统一的 FileLike trait 抽象各类文件描述符，结合 FlattenObjects 稀疏数组管理 FD 表，支持管道、eventfd、signalfd、pidfd 和 epoll 等丰富的文件类型。信号子系统支持完整的实时信号队列、信号掩码、sigreturn trampoline 及 signalfd，实现了与 Linux 高度一致的信号处理链路。在进程管理方面，项目提供了细粒度的 clone 标志支持、COW 地址空间复制、futex 同步及 robust list 处理。内存管理实现了 mmap/munmap 的多种映射方式与按需填充，并通过 VmPtr/VmMutPtr 安全抽象提供零开销的用户态内存访问。项目支持 RISC-V64、LoongArch64、x86_64 和 AArch64 多种架构，并通过 patch 机制精准替换上游组件，在保持与 ArceOS 框架兼容的同时实现深度定制。其将组件化 unikernel 框架逆向重组为隔离型宏内核的设计思路，以及 scope 驱动的资源管理方式，展现了 Rust 在系统软件领域出色的工程能力与安全抽象价值。
