Anemone OS 是一个采用 Rust 语言实现的高完成度宏内核项目，定位为 Linux 兼容的通用操作系统内核。它通过 trait 驱动的架构抽象同时支持 RISC-V 64 和 LoongArch 64 双架构，以约 85,000 行 Rust 代码实现了进程管理、虚拟文件系统、内存管理、信号处理、设备模型与驱动、同步原语等核心子系统。该项目实现了约 144 个系统调用，涵盖文件 I/O、内存映射、futex、capability 和 prctl 等关键接口，在 VFS 层提供了 ramfs、procfs、devfs、ext4 及匿名管道等五种文件系统，并具备完整的挂载系统与路径解析能力。其亮点包括：使用过程宏自动收集系统调用处理器与分级初始化函数，避免手工维护分发表；采用双向映射结构管理文件描述符，实现常数级查找；编译期构建引导页表，缩短启动路径；以及将伙伴分配器、设备树解析等组件拆分为独立可测试的子 crate。整体工程采用 kconfig 风格的配置系统与 xtask 构建工具链，并内建 KUnit 单元测试框架，体现出规范的现代 Rust 工程实践。尽管调度器仅实现了轮转与空闲调度类、尚未集成网络栈，但在同类 Rust 宏内核中，Anemone OS 在子系统广度、Linux 兼容性以及代码质量方面均展现出较高水准。
